# 西宮合氣道同好会 公式サイト

## プロジェクト概要
地域の合氣道同好会の公式ホームページ。主目的は新規会員の獲得。
「稽古に来てみたい」という気持ちを引き出すサイトを目指す。

## 技術スタック
- Next.js 14（App Router）/ TypeScript / Tailwind CSS
- SSG（静的サイト生成）前提
- デプロイ先：Vercel（無料枠）

---

## 自律開発ワークフロー

**ブリーフ（コンセプト）を受け取ったら、確認なしに以下のフェーズを順番に実行すること。
エラーが出たら自己解決して次へ進む。すべて完了したら完了レポートを出力する。**

### Phase 1：要件定義（docs/requirements.md に記録）
- ブリーフを分析し、ターゲット層・訴求ポイントを明確化
- 必要ページと各ページの役割を決定
- CTA の設計（何をしてもらいたいか）

### Phase 2：市場調査（docs/market-research.md に記録）
- Web 検索で類似サイト 3〜5 件を調査
  - 検索例：「合氣道 同好会 ホームページ」「武道 地域クラブ サイト」「aikido club website japan」
- 効果的なデザインパターン・コピーのパターンを特定
- 差別化ポイントを整理

### Phase 3：設計（docs/design.md に記録）
- ページ構成と各ページのコンテンツ設計
- コンポーネント一覧
- カラーパレット（Tailwind クラス名で指定）・フォント方針

### Phase 4：実装
- プロジェクト初期化：
  ```bash
  npx create-next-app@latest . --typescript --tailwind --app --no-git --yes
  ```
- ページごとに実装し、各ページ後に `npm run build` でエラーチェック
- エラーがあれば即修正してから次へ

### Phase 5：品質確認
- ビルドエラーゼロを確認
- モバイルファースト（スマホ表示を最優先）
- 文字サイズ 16px 以上、コントラスト比 WCAG 2.1 AA 以上
- 問い合わせ・入会への動線が明確

### Phase 6：完了レポート（docs/report.md に出力）
- 作成ページ一覧とファイルパス
- デザインの意図
- 未解決の課題・改善候補
- 確認方法：`npm run dev` → `http://localhost:3000`

## デザイン原則
- 落ち着き・誠実さ・身体の動きの美しさを表現
- カラーベース：白・紺・金（Tailwind: white / slate-800 / amber-500）
- 余白と写真で魅せる（過度な装飾は避ける）
- 「ここなら始められそう」と感じるやさしい雰囲気
- 画像は placeholder.com を使用（本番では差し替え）

## 標準ページ構成

| ページ | パス | 役割 |
|---|---|---|
| トップ | `/` | ファーストビュー・概要・CTA |
| 合氣道について | `/about` | 理念・特徴・他武道との違い |
| 活動紹介 | `/activity` | 稽古内容・スケジュール・会場 |
| 入会案内 | `/join` | 費用・流れ・FAQ |
| お問い合わせ | `/contact` | フォーム or メールリンク |

## コーディング規則
- コンポーネント：`src/components/`
- ページ：`src/app/`（App Router）
- 画像：`public/images/`
- コメントはロジックが自明でない場合のみ
- 型定義は `src/types/` に集約
